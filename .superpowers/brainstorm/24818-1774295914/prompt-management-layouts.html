Quick Comparison

A

Top-Down

Pro: Metrics always visible alongside editor — no tab switching to compare.
Con: Cramped — editor and metrics compete for horizontal space. Gets worse on smaller screens.

B

Tab-Based (Recommended)

Pro: Each view (versions, compare) gets full width. Clean separation. Matches your existing UI patterns.
Con: Need to switch sub-tabs to see metrics — not visible while editing.

C

List-Detail

Pro: All prompt types + versions visible in one tree. Inline metrics per version.
Con: No side-by-side comparison view. Tree can get long with many versions. Comparison needs a separate interaction (select two versions).
