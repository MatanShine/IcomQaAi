Variable Marking Inside Editor — Close-up

Variables are detected and rendered with a distinct background so the user knows they'll be replaced at runtime

Editor Close-up — Variable Detection
Available topics in our knowledge base: ​{question_titles_text}​variable - injected at runtime Context from knowledge base: ​{kb_context}​ <-- yellow = appears 2x in prompt Previous search queries: ​{previous_queries}​ Remember: answer only from ​{kb_context}​ or history. Not a variable: {some_random_text} <-- not highlighted, not a known variable

How variable detection works:

  • Only known variables (registered in the system) get highlighted — random {text} is left plain
  • In-editor: variables get a colored background pill matching their usage count color
  • Blue pill = used exactly once
  • Yellow pill = used more than once (the count badge shows exact number)
  • Variables below editor show the same colors + a count badge (hover for tooltip)
  • Clicking a variable in the list below could scroll/highlight its occurrences in the editor
